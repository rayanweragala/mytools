# Contributing to Webhook Mock Lab

We appreciate your interest in contributing! Whether it's a bug report, a feature request, or a code contribution, your help is valued.

## 🚀 How to Contribute

### Reporting Bugs
- Search existing issues to ensure the bug hasn't already been reported.
- Use the **Bug Report** template when opening a new issue.
- Provide clear steps to reproduce the issue.

### Requesting Features
- Use the **Feature Request** template.
- Explain the "Why" behind the feature.

### Code Contributions
1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature-name` or `bugfix/issue-id`.
3. Make your changes and ensure they follow the project's style.
4. Update the documentation if necessary.
5. Commit your changes with clear, descriptive messages.
6. Push to your fork and submit a Pull Request.

## 🛠️ Local Development

```bash
# Clone your fork
git clone https://github.com/your-username/webhook-mock-lab.git
cd webhook-mock-lab

# Install dependencies
npm install

# Run in development mode
npm run dev
```

## 📝 Commit Message Guidelines
- Use clear and concise subject lines.
- Reference issues if applicable (e.g., `Fixes #123`).

## ⚖️ Code of Conduct
Please be respectful and professional in all interactions.

Thank you for contributing!
